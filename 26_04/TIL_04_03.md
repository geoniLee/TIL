# TIL 4.03
<h3>알고리즘 문제 풀이</h3>
<h4>2473 세 용액</h4>

* 알고리즘
    * 정렬
    * 두 포인터
* 아이디어
    1. 두포인터를 사용하기 위해 입력된 용액을 정렬
    2. 첫번째 용액을 -1을 곱해 저장하고, 두번째 세번째 용액을 두 포인터로 비교
    3. 첫번째 용액을 저장한 값과 두번째/세번째 용액의 합을 비교
* 회고
    1. 용액의 특성(-10억 ~ 10억) 이므로 첫번째 용액/ 두번째, 세번째 용액의 합으로 비교하는 과정에서 int 범위를 벗어나지 않을 것으로 판단하여 int 사용
    2. 이후 세 용액의 합을 계산하고 절댓값으로 저장하는 과정에서 결과(-30억 ~ 30억)가 int 범위를 넘어가는 점을 간과함

<h4>2805 나무 자르기</h4>

* 알고리즘
    * 이분 탐색
    * 정렬

* 아이디어
    1. 0 ~ 최대 높이의 나무을 범위로 잡아 중간값을 사용해 이분탐색
    2. 오름 차순으로 정렬한 뒤 전체 나무를 순회하며 mid - 나무의 높이가 0 이하가 되면 mid - 나무 들의 합과 m을 비교
    3. 합과 m이 같으면 mid을 출력하고 종료
    4. 합과 m이 같지 않으면 새롭게 범위를 탐색하여 이분 탐색을 진행
    5. 합이 m보다 큰 경우 최대 높이를 갱신
