<h3>알고리즘 문제 풀이</h3>
<h4>33627 네모네모</h4>

* 알고리즘
    * 그리디 알고리즘
    * DP(다이나믹 알고리즘)
* 아이디어
    1. 입력된 x을 기준으로 배치 가능한 공간을 구해 vector로 저장
    2. dp로 입력된 a에 대해 모든 배치가 가능한 최소 공간과 남은 칸을 저장
    3. 현재 블록이 배치 가능한 첫번째 공간을 탐색
    4. 탐색된 공간에서 현재 블록의 크기 만큼 감소
    5. 현재 블록을 다음 공간에 배치해도 모든 블록이 배치 가능한지 확인
        1. 배치 가능하다면 현재 블록은 항상 배치되는 영역을 만들수 없으니 다음 블록을 탐색
    6. 배치 불가능한 경우
        1. 마지막 블럭이라면 다음 블록이 없으니 블럭의 크기 - 남은 공간의 크기을 음수가 나오지 않도록 보정한 값을 출력값에 합산
        2. 마지막 블럭이 아닌 경우 다음 블럭을 가져와 현재 공간이 아닌 다음 공간으로 빠질 수 있는지 판단
        3. 다음 공간으로 빠질 수 있는 블록이 나올 때까지 반복
        4. 블럭의 크기 -(남은 공간의 크기 - 다음 공간으로 빠질수 없는 블록의 크기들)을 음수가 나오지 않도록 보정하여 출력 값에 합산
    7. 다음 블록 또한 현재 공간에 들어갈수 없는 경우
        1. 블럭의 크기 - 남은 공간의 크기을 음수가 나오지 않도록 보정한 값을 출력값에 합산

* 회고
    1. 항상 채워지는 영역을 계산할 때 다음 블록 하나만 고려하려고 함 
        * 해당 공간에 다른 블록이 더 들어와야 하는 경우를 고려하지 않음
    2. 배치 성공 여부를 시뮬레이션 돌릴려고 함 
        * 서브태스크 특정 케이스에서 매우 큰 입력 값을 요구하기에 시간 제한 및 메모리 제한에 걸렸을 것
    3. 문제 난이도 결정 의견 확인 이후
        1. 좌측 그리디 + 우측 그리디
        2. 좌측 그리디 + 우측 그리디의 중복된 부분을 카운트 하여 출력
