<h4>언리얼 엔진 Actor의 라이프 사이클 이해하기</h4>

* UE_Log(log 카테고리,로그 수준, TEXT(""))
    * 로그 수준
        * Display: 일반적인 실행 흐름이나 상태 확인 메시지(흰색)
        * Waring: 예상치 못한 동작이나 잠재적인 문제(노란색)
        * Error: 즉시 수정이 필요한 심각한 문제(빨간색)
    ---
    * 고유한 카테고리 정의
        * 헤더
            > DECLARE_LOG_CATEGORY_EXTERN(이름, 기본 로그 수준, CompileTimeVerbosity)

            * 기본 로그 수준: 어디까지 로그를 출력할지 지정(Fatal - Error - Warning - Display - Log - Verbose - VeryVervose)
                * ini, 콘솔 명령, 실행 옵션 등으로 변경 가능
            * CompileTimeVerbosity: 최대 로그 수준
            * 보통 이러한 로그 카테고리는 별도의 공용 헤더에서 관리함
        * Cpp
            * DEFINE_LOG_CATEGORY(이름)

* LifeCycle
    * 생성자: 메모리에 생김 딱 한번 호출
    * PostInitializeComponent() - 컴포넌트가 완성된 직후 호출, 컴포넌트끼리 데이터 주고 받기, 상호작용
    * BeginPlay() - 배치(Spawn) 직후
    * Tick(float DeltaTime) - 매 프레임마다 호출
    * Destroyed() - 삭제되기 직전에 호출됨
    * EndPlay(const EEndPlayReason::Type 이름) - 게임 종료, (Destroyed()), 레벨 전환

<h4>Tick 함수로 Actor의 Transform 조정하기</h4>

* 위치: 월드에서 어느 지점에 있는지를 나타냄
* 회전: 어느 방향을 바라보는지, 어떤 각도로 기울어져 있는지를 나타냄
    * Pitch 앞뒤 방향의 기울어짐(x축을 축으로 하는 회전)
    * Yaw: 좌우 방향 회전(y축을 축으로 하는 회전)
    * Roll: 좌우로 기울어지는 회전(z축을 축으로 하는 회전)
* 스케일: 액터의 크기 비율

* FTransfrom: 위치, 회전, 스케일을 하나로 묶어 효율적으로 관리하기 위한 구조체
    * Transform - FVector(x,y,z)
        * **SetActorLocation(FVector())**: Lotation 설정
        * **GetActorLocation()**: (FVector) Actor의 Location을 받아옴

        * **AddActorLocalOffset(FVector())**: FVector 값 만큼 Location 이동(로컬 좌표)
        * **AddActorWorldOffset(FVector())** : FVector 값 만큼 Location 이동(월드 좌표)

    * Rotation - FRotation(Pitch,Yaw,Roll)
        * **SetActorRotation(FRotator())**: Ratation 설정
        * **GetActorRotation()**: (FRotator) Actor의 Rotation을 받아옴

        * **AddActorLocalRotation(FRotator())**: FRotator 값 만큼 이동(로컬 좌표)
        * **AddActorWorldRotation(FRotator())**: FRotator 값 만큼 이동(월드 좌표)

    * Scale3D - FVector(x,y,z)
        * SetActorScale3D(FVector()): Scale 설정
        * GetActorScale(): (FVector) Actor의 Scale을 받아옴
