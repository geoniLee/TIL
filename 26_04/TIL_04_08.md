<h3>Unreal Engine으로 3D 게임 개발</h3>
<h4>클래스와 리플렉션 시스템 활용하기</h4>

* Cpp Vs 블루 프린트
    * CPP
        * 게임 플레이 (핵심)로직
    * 블루 프린트
        * UI
        * 시네마틱
        * 프로토타이핑
        * 애니메이션

* 리플렉션 시스템
    * 리플렉션 시스템: Cpp 클래스의 변수 및 함수 정보를 엔진 내부의 메타데이터 형태로 저장하고, 이를 에디터나 블루 프린트에서 활용할 수 있게 만들어주는 기술
    * include "Name.generated.h"
        * 리플렉션 시스템을 지원하는 헤더 파일
        * 항상 include 가장 아래에 위치
    * UCLASS()
        * 클래스를 리플렉션 시스템에 등록해주는 매크로
        * 주요 옵션
            * Blueprintable: 블루프린트에서 상속 가능한 클래스로 만듭니다.
            * NotBlueprintable: 블루프린트에서 이 클래스를 상속할 수 없도록 합니다
            * BlueprintType: 블루 프린트에서 변수나 참조로 사용할 수 있게 합니다.
                * 이 옵션만 있으면 상속은 허용되지 않고 참조만 가능합니다.
    * GENERATED_BODY()
        * 리플렉션 데이터를 자동으로 생성하기 위해 언리얼 엔진이 사용하는 매크로
    * UPROPERTY()
        * 지정자를 작성해 에디터에서의 표시 여부나 Blueprint 접근성, 읽기/쓰기 권한 등을 자세히 설정할 수 있음
        1. 편집 가능 지정자
            * VisibleAnyWhere: 읽기 전용, 수정 불가능
            * EditAnyWhere: 클래스 기본값, 인스턴스 모두에서 수정 가능
            * EditDefaultsOnly: 클래스 기본값에서만 수정 가능
            * EditInstanceOnly: 인스턴스 에서만 수정 가능
        2. Blueprint 접근성 지정자
            * BlueprintReadWrite: Blueprint 그래프에서 Getter/Setter 사용 가능
            * BlueprintReadOnly: Blueprint에서 Getter 사용 가능
        3.  Category 지정자
            * Rotation 아래에 표시됨
            >ex) Category = "Item|Components"
        4. 메타 옵션 지정자
            * meta=(Clamp="0.0"): 에디터에서 변수 입력 시 최소값을 제한 가능
            * meta=(AllowPrivateAccess = "true"): 해당 멤버가 private로 선언되어 있어도, 에디터/Blueprint에서 접근할 수 있도록 함
    * UFUCTION()
        * 함수를 블루프린트에서 호출할 수 있도록 등록
        * 주요 지정자
            * BlueprintCallable: 이벤트 그래프에서 호출 가능한 함수로 만듦
            * BlueprintPure: Getter 역할만 수행
            * BlueprintImplementableEvent
                * 함수의 선언 - Cpp
                * 함수의 구현 - Blueprint
