<h3>언리얼 엔진 Cpp 심화 강의</h3>
<h4>리플렉션/UHT/UBT/CDO</h4>

* 좋은 코드
    * 함수, 변수 이름 등에서 기능을 예측할 수 있어야 한다
    * 중복 코드 -> 중복 코드는 하나만 수정해도 되도록 모아놓아야 함
    * 긴 함수 -> 짧은 함수로 나누어서

* 리플렉션
    * 리플렉션: 실행 중에 프로그램이 자기 자신의 구조를 인지하고 조작할수 있도록함
    * 역할
        * 런타임 정보 확인
        * 에디터 연동
        * 가비지 컬렉션 지원

    * 가비지 컬렉션(UObject)
         * 스마트 포인터(TWeakObjectPt)
    * 핵심 매크로
        * UCLASS: 클래스를 언리얼에 등록
        * UPROPERTY: 변수를 에디터에 노출
        * UFUCTION: 함수를 블루프린터에 등록
* UHT
    * UHT(언리얼 헤더 툴)와 메타 데이터: 컴파일 전 마법사- 코드를 에디터가 이해할 수 있게 번역
        * UCLASS, UPROPERTY, UFUNCTION 메타 데이터 수집
* UBT
    * UBT(언리얼 빌더 툴)
        * 가장 먼저 실행되는 빌더 도구
* CDO
    * CDO(Class Default Object): UObject 클래스 마다 하나씩 존재하는 기본값 보관소
        > CDO는 모든 인스턴스의 '원본 템플릿
    * 생성 시점
        * 모듈 로딩 단계에서 생성
        * UBT -> UHT -> 컴파일 -> 엔진 커짐
        * 생성자 딱 한 번만 호출
    * 초기화 및 복제
    * 최적화 원리
        * 메모리 절약 - 공통 기본값 CDO 하나에만 저장
        * 빠른 초기화 - 런타임 체크 즉각 가능 
        * 델타 직렬화 - 차이점만 기록, 전송
* 전체 빌드 타임라인
    > UBT 실행 -> UHT 처리(generated.h 생성) -> VS 컴파일(#include generated.h) -> 리플렉션
