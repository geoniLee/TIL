<h3>Unreal Packaging</h3>
<h4>Lambda Function</h4>

* Lambda Function 자체가 불안정한 것은 아니지만, Unreal에서는 Timer, Delegate, AsyncTask, 비동기 로드 롤백 등 지연 실행 구조에서 자주 사용되기 때문에 객체 수명 관리에 주의가 필요함

* 지연 실행되는 Lambda가 UObject, Actor, Component, Widget, this 등을 캡쳐하면 GC, Destroy/EndPlay, UI제거, Level 전환, 로드 타이밍, 초기화/종료 순서 차이 등으로 실행 전에 무효화 될 수 있다.

* 특히 이 수명 문제는 Packaging 빌드에서는 최적화, Cooked Asset 환경의 GC 타이밍 차이로 인해 더 쉽게 드러날 수 있다.

<h4>Unreal Insights</h4>

* Unreal Insights : Unreal Engine에서 제공하는 Trace 기반 성능 분석 도구

    게임이나 에디터가 실행되는 동안 CPU, GPU, Frame, LoadTime, File I/O, Log, Bookmark 같은 이벤트를 기록하고, 그 기록을 .utrace 파일로 저장한 뒤 시각적으로 분석할 수 있게 해준다.

* 주요 구성요소
    * Trace Events : 실행 중 발생하는 성능 / 상태 관련 이벤트 데이터
    * Unreal Trace Server : 실행 중인 애플리케이션 에서 온 Trace를 기록하고 저장함
    * Timing Insights : 저장된 Trace 세션 데이터를 분석/시각화

* Trace Channel

    Unreal Insights는 필요한 Trace Channel을 활성화 해서 데이터를 수집한다.

    * 필요한 채널을 켜지 않으면 해당 정보는 .utrace에 기록되지 않는다

* 기능

    프레임 단위로 어디서 시간이 많이 걸렸는지 보는 것

    * Timing Insights : UE Trace Session을 분석하는 메인 창
    * CPU/GPU 트랙
    * 특정 프레임에서 처리 시간이 튄 지점 확인
    * 어떤 이벤트가 오래 걸렸는지 확인
    * 어떤 스레드/코어에서 어떤 작업이 실행되었는지 확인
    * 호출자/피호출자 관계 확인
    * Task 실행 경로 확인
    * Context Switch 확인

* 실행 옵션 / 콘솔 명령 예시
    
    명령줄
    ```
    -trace=cpu,gpu,frame,bookmark,loadtime,file
    ```
    <br>

    콘솔 명령 예시
    ```
    Trace.Status <br>
    Trace.Start <br>
    Trace.Stop <br>
    Trace.File <br>
    ```


* 사용 흐름
    1. UnrealInsights.exe 실행
    2. 프로젝트 실행
    3. Session Browser에서 LIVE 세션 확인
    4. Trace 열기
    5. Timing Insights에서 프레임/스레드/이벤트 분석

* 분석 방법
    1. Packaged exe를 Trace 옵션과 함께 실행
    2. 분석하고 싶은 상황까지 플레이
    3. Bookmark 또는 ScreenShot으로 지점 표시
    4. Trace 종료
    5. Unreal Insights에서 .utrace 열기
    6. Timing Insights에서 해당 시간대 확대
    7. GameThread / RenderThread / GPU / LoadTime / File I/O 확인
