<h3>알고리즘 문제 풀이</h3>
<h4>등굣길</h4>

* 알고리즘
    * DP(다이나믹 프로그래밍)

* 아이디어
    1. 1,1 부터 순회하며 [x-1][y], [x][y-1]의 값을 더함
    2. 각 값을 더할 때 overflow방지를 위해 1000000007로 나눈 나머지를 저장
    3. 목표지점 n,m 의 좌표를 출력

<h3>언리얼 심화 라이브 세션</h3>
<h4>오브젝트 풀링</h4>

* Unreal 사용 예

    * 각 오브젝트트에 추적 컴포넌트를 부착하여 관리

    * 구조
        * Pool -> Actor(Manager)
        * PooledObject -> Component(추적용도)
        * PoolObjectData -> None(파라미터용)

    * `SpawnParams.NameMode` : `SpawnParams.Name`에 저장한 이름을 Spawn 시 어떻게 처리할 지 결정
        * `Request`: 요청한 이름을 사용하려고 시도

            같은 이름을 사용 중이면 해당 이름을 기반으로 사용 가능한 이름을 자동 생성
        * `Required_returnNull`: 요청한 이름을 반드시 사용(실패 - nullptr)
        * `Required_ErrorAndReturnNull`: 요청한 이름을 반드시 사용(실패 - Error 기록 후 nullptr)
        * `Required_Fatal`: 요청한 이름을 반드시 사용해야함(실패 - fatal)
    * `SpawnParams.SpawnCollisionHandlinhOverride` : Spawn 위치에서  겹쳤을때 처리 방식
        * `Undefined`: 별도 Override를 하지 않고 기본 설정을 따름
        * `AlwaysSpawn`: 항상 스폰
        * `AdjustIfPossibleButAlwaysSpawn`: 가능하면 충돌하지 않는 위치로 보정(항상 Spawn)
        * `AdjustIfPossibleButDontSpawnIfColliding`: 가능하면 충돌하지 않는 위치로 보정(성공 시에만 Spawn)
        * `DontSpawnIfColliding`: Spawn 위치에 충돌이 있으면 Spawn하지 않음

    * 사용 순서
        1. Spawn
            1. Actor 생성
            2. 컴포넌트 만들기
            3. 등록
            4. 컴포넌트 초기화
            5. 액터 설정(Hidden, EnableCollsion, Tick, 자식 오브젝트로 지정 등)
            5. pool에 컴포넌트 저장
