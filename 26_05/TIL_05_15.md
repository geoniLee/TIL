<h4>거스름돈</h4>

* 알고리즘
    * DP

* 아이디어
    1. dp[0]을 1로 초기화
    2. money을 순회하며, dp[x] 에 dp[x - c]을 x 가 x<= 인 동안 더함
    3. 각 연산에 1000000007로 나눈 나머지 저장
    4. dp[n]반환

<h3>언리얼 심화 라이브세션</h3>
<h4>Soft Object / Class Reference</h4>

* 소프트 레퍼런스 & 하드 레퍼런스
    * 하드 레퍼런스(UObject* TObjectPtr, TSubclassOf)

        * 참조 대상이 함께 로드될 가능성이 높음
        * 프로젝트 규모가 커질 수록 불필요한 에셋까지 올라올 수 있음

    * 소프트 레퍼런스 
        * 에셋의 경로를 통해 참조 -> 필요한 시점에 동기/비동기로 로드

* 소프트 오브젝트 레퍼런스

    TSoftObjectPtr<T>

    * 참조 대상 
        * 에셋 인스턴스
        * 오브젝트

* 소프트 클래스 레퍼런스

    TSoftClassPtr<T>

    * 참조 대상
        * 스폰하거나 생성할 클래스

* LoadSynchronous -> 동기로드
    * 소프트 레퍼런스가 가리키는 에셋을 즉시 로드
* RequestAsyncLoad(경로) -> 비동기 로드
    * 에셋 경로를 기반으로 에셋을 비동기로 로드
    * 에셋 로드가 끝나면 등록해둔 콜백/델리게이트가 호출됨

<h4>데이터 에셋</h4>

* DataTable

    UStruct를 기반으로 여러 행의 데이터를 표 형태로 관리하는 방식
    * 비슷한 구조의 데이터를 많이 관리할 때 적합
    * RowName을 통해 특정 데이터를 찾을 수 있음
* DataAsset

    하나의 에셋 파일안에 특정 데이터 묶음을 저장하는 방식
    * 하나의 객체 단위 데이터를 사용할 때 적합

* PrimaryDataAsset

    Asset Manager가 직접 관리할 수 있는 DataAsset
    * Primary Asset Id를 통해 에셋을 식별할 수 있음
    * Asset Manager를 통해 수동 로드/언로드 관리가 가능함
    * 단순 데이터 저장보다, 에셋 로딩/관리 체계와 연결할 때 사용

    * meta = (AssetBundles = "BundleName")
        * PrimaryDataAsset 내부의 Soft Reference를 특정 Bundle에 포함시키기 위한 메타 데이터
        * AssetManager가 PrimaryDataAsset을 로드할 때, 어떤 Soft Reference 까지 함께 로드할지 구분하는 용도로 사용
