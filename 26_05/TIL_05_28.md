<h3>디자인 패턴</h3>
<h4>오브젝트 풀</h4>

다수의 오브젝트를 효율적으로 관리하고, 런타임 성능을 안정화하기 위한 디자인 패턴

* 빈번한 오브젝트 생성과 삭제는 메모리 할당/해제 비용, 런타임 성능 저하, 프레임 드랍등을 유발함

    -> 위와 같은 문제를 줄이기 위해 오브젝트를 *미리 생성* 하거나 *최초 사용 시 생성*한 뒤 풀에 보관하고, 이후에는 재사용하는 방식으로 동작 

* 핵심 요소
    1. 미리 생성된 오브젝트의 저장소(풀)

        필요한 수만큼 오브젝트를 생성하여 풀에 저장<br>
        일반적으로 List or Stack 사용
    2. 오브젝트 활성화와 비황성화

        사용할 때 -> 활성화
        사용이 끝났을 때 -> 파괴하지 않고 비활성화 하여 풀에 반납
    3. 재사용을 위한 초기화

        반납 시 이전 상태를 초기화 하고, 다시 꺼낼 때 사용 목적에 맞게 필요한 값을 설정함
