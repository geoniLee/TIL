<h3>Ch03 TeamProject</h3>
<h4>위치 보정</h4>

* 상황
    1. Enemy위치에 건물이 설치 되었을 때 Z값이 올라감
    2. 위치 보정은 가까운 NavMesh위치로 이동시키는데 올라간 Z값 때문에 위쪽 NavMesh로 보정됨
    3. 공중 NavMesh는 Player나 일반적인 Enemy의 동작을 제한하지 않기 위해 Pawn - Ignore / 탈것 - Block으로 설정함
    4. 위쪽에 보정된 Enemy는 무시되어 아래로 떨어지고 NavMesh 밖에 위치하게 되어 움직이지 않음

* 해결 방안
    * 보정과정에서 Z 값을 Spawn때와 마찬가지로 OffSet을 반영해서 찾도록 함(지상 - 0 / 공중 - 250)

<h4>UI 갱신 로직 변경</h4>

* 상황
    1. UI, HUD 갱신을 NativeTick에서 진행하여 매 Tick마다 호출하며 갱신 시도
    2. Core관련된 부분 초기화를 델리게이트를 받아 갱신하도록 변경
        1. Core 생성 시 Broadcast을 보내 Core 갱신
        2. Core의 체력 변동 시 Broadcast을 보내 UI에 반영
    3. 자원 관련해서 델리게이트가 구현되어 있어서 Tick에서 처리하는 기존 로직을 지우고, 델리게이트로 변경

<h4>Player 공격 Enemy</h4>

* 공격 조건
    * 이동 중 탐지 범위 내에 들어오면 공격
    * 탐지는 이동 중에만 활성화
    * Core / Tower 공격 중에는 탐지하지 않음

* 공격 로직
    * 탐지 범위 내 Player가 감지되면 정지
    * 공격 딜레이 이후 공격 범위(원형) 만큼 충돌 감지
        * Player -> 밀림 + Slow + Damage
        * Tower -> 지정된 시간 동안 무력화 (구현 완료 시 연결 예정) 

<h4>Player 피해, Debuff 구현</h4>

* Enemy가 Player 공격 피해를 입히기 위해 TakeDamage 생성
* Slow Debuff 구현
    * 이동 속도에 SlowRate을 적용
    * TimerHandler을 사용해 지정된 시간 이후에 원래 속도로 복원

<h4>UI 갱신 오류 수정</h4>

1. CoreHP 미 반영
    * 상황
        * CoreBase가 갱신되지 않으며 Pause -> resume 할 때 한번 씩 갱신

    * 원인
        1. Pause 하는 과정에서 NativeDestruction에서 Core을 null로 하지 않음
        2. Core가 저장된 Core와 동일하여 UI에 반영되는 부분이 실행되지 않음

2. HP UI percent 반영 안됨
    * 상황
        1. 시작 시 HP 100%
        2. 공격을 받게 되면 HP 0%
    
    * 기존 구현된 코드에서 HP / MaxHp을 static_cast<int>로 HP_Bar을 갱신
        * 0이나 1로 넘어감
    * static_cast<int>을 제거하여 수정
