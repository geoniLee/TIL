## Animation Montage 기반 Gameplay Ability 흐름

### 1. `UAbilityTask_PlayMontageAndWait`

Gameplay Ability에서 **Animation Montage를 재생하고, 몽타주 상태에 따라 콜백을 받는 Ability Task**이다.

주요 델리게이트:

* `OnCompleted`

  * 몽타주가 정상적으로 끝까지 재생되었을 때 호출된다.
* `OnBlendOut`

  * 몽타주가 종료되면서 블렌드 아웃을 시작할 때 호출된다.
* `OnInterrupted`

  * 다른 몽타주나 외부 처리에 의해 재생이 중단되었을 때 호출된다.
* `OnCancelled`

  * Ability 취소 등의 이유로 몽타주가 취소되었을 때 호출된다.

> `OnBlendIn`은 `UAbilityTask_PlayMontageAndWait`의 기본 델리게이트가 아니다.

---

### 2. `ReadyForActivation()`

Ability Task를 생성한 뒤 실제 동작을 시작시키는 함수이다.

```cpp
MontageTask->ReadyForActivation();
```

일반적인 순서:

```cpp
UAbilityTask_PlayMontageAndWait* MontageTask =
    UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
        this,
        NAME_None,
        Montage
    );

MontageTask->OnCompleted.AddDynamic(this, &ThisClass::OnMontageCompleted);
MontageTask->OnInterrupted.AddDynamic(this, &ThisClass::OnMontageInterrupted);
MontageTask->OnCancelled.AddDynamic(this, &ThisClass::OnMontageCancelled);

MontageTask->ReadyForActivation();
```

`ReadyForActivation()`을 호출하지 않으면 Task가 생성되어도 몽타주 재생과 이벤트 대기가 시작되지 않을 수 있다.

### 3. Gameplay Event 대기

`WaitGameplayEvent`는 `UAbilityTask_PlayMontageAndWait`의 기능이 아니라 별도의 Ability Task인 `UAbilityTask_WaitGameplayEvent`를 사용한다.

```cpp
UAbilityTask_WaitGameplayEvent* EventTask =
    UAbilityTask_WaitGameplayEvent::WaitGameplayEvent(
        this,
        EventTag
    );

EventTask->EventReceived.AddDynamic(
    this,
    &ThisClass::OnGameplayEventReceived
);

EventTask->ReadyForActivation();
```

주요 흐름:

```text
Animation Montage 재생
        ↓
AnimNotify 발생
        ↓
Gameplay Event 전송
        ↓
UAbilityTask_WaitGameplayEvent가 이벤트 수신
        ↓
EventReceived 콜백 실행
        ↓
실제 공격·콤보·투사체 생성 등의 로직 실행
```

정확한 델리게이트 이름은 일반적으로 `EventReceived`이다.

---

### 4. `AnimNotify`

Animation Montage의 특정 프레임에서 이벤트를 발생시키기 위한 기능이다.

일반적인 `UAnimNotify`는 다음 함수를 오버라이드한다.

```cpp
virtual void Notify(
    USkeletalMeshComponent* MeshComp,
    UAnimSequenceBase* Animation,
    const FAnimNotifyEventReference& EventReference
) override;
```

`Notify()`가 호출되는 시점에 다음과 같은 처리를 할 수 있다.

* Gameplay Event 전송
* 공격 판정 시작
* 투사체 생성
* 사운드 재생
* 파티클 생성
* 콤보 입력 가능 상태 활성화

멀티플레이 환경에서는 AnimNotify에서 직접 핵심 게임 상태를 변경하기보다, **서버 권한을 확인하거나 Gameplay Event를 통해 Ability 로직으로 전달하는 방식**이 적합하다.

---

### 5. 콤보 입력 대기

`SetupWaitComboInputPress()`는 Unreal GAS의 기본 함수가 아니라 프로젝트에서 별도로 구현한 사용자 정의 함수로 보인다.

일반적으로 내부에서는 `UAbilityTask_WaitInputPress`를 생성한다.

```cpp
void UMyGameplayAbility::SetupWaitComboInputPress()
{
    UAbilityTask_WaitInputPress* InputTask =
        UAbilityTask_WaitInputPress::WaitInputPress(this);

    InputTask->OnPress.AddDynamic(
        this,
        &ThisClass::OnComboInputPressed
    );

    InputTask->ReadyForActivation();
}
```

콤보 입력 처리 예시:

```text
첫 번째 몽타주 재생
        ↓
콤보 입력 허용 구간 AnimNotify 발생
        ↓
WaitInputPress Task 생성
        ↓
플레이어 입력 감지
        ↓
다음 콤보 예약
        ↓
다음 몽타주 Section으로 이동
```

### 6. `K2_EndAbility`

Blueprint 또는 C++에서 현재 Gameplay Ability를 종료하는 기능이다.

Blueprint 노드 이름은 일반적으로 `K2_EndAbility`이며, C++ 내부에서는 주로 `EndAbility()`를 사용한다.

```cpp
EndAbility(
    CurrentSpecHandle,
    CurrentActorInfo,
    CurrentActivationInfo,
    true,
    false
);
```

몽타주가 정상 종료되거나 취소·중단되었을 때 Ability가 계속 활성화된 상태로 남지 않도록 호출해야 한다.

```cpp
void UMyGameplayAbility::OnMontageCompleted()
{
    K2_EndAbility();
}

void UMyGameplayAbility::OnMontageInterrupted()
{
    K2_EndAbility();
}
```

## 전체 구조

```text
Gameplay Ability 활성화
        ↓
PlayMontageAndWait Task 생성
        ↓
몽타주 관련 델리게이트 등록
        ↓
ReadyForActivation()
        ↓
Animation Montage 재생
        ↓
AnimNotify 발생
        ↓
Gameplay Event 전송
        ↓
WaitGameplayEvent의 EventReceived 실행
        ↓
공격 또는 콤보 로직 처리
        ↓
몽타주 완료·취소·중단
        ↓
K2_EndAbility
```

## 용어 수정

* `K2_ENDAbility` → `K2_EndAbility`
* `EventRecieve` → `EventReceived`
* `OnBlendIn/Out` → 기본 `PlayMontageAndWait`에서는 주로 `OnBlendOut`
* `WaitGameplayEvent` → `PlayMontageAndWait` 내부 기능이 아닌 별도의 `UAbilityTask_WaitGameplayEvent`
* `SetupWaitComboInputPress()` → GAS 기본 API가 아닌 프로젝트 사용자 정의 함수로 추정
