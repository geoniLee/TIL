# TIL 7.20
<h4>ch04 팀프로젝트</h4>

* 아이템 사용 조건 개선

    래그돌 상태처럼 아이템을 사용할 수 없는 상태에서 입력 Release가 들어와도 아이템이 최종 실행되지 않도록 수정
    
    * 아이템 Ability 실행 전 `DoesAbilitySatisfyTagRequirement`s로 현재 상태 태그 조건을 재검사
    * Pressed -> 래그돌 진입 -> Released 흐름에서 아이템이 뒤늦게 사용되는 문제 방지
    * 실행 불가 상태면 Ability를 종료하고 아이템 소비 처리로 넘어가지 않도록 수정
    * 아이템 슬롯 컴포넌트에서도 Release 시점에 Ability 태그 조건을 확인하도록 보강

* 아이템 사용 불가 상태 태그 설정
    
    각 아이템 Ability 에셋에 “사용 불가 상태”를 판단하기 위한 태그 조건을 지정

* UI - Character 선택 갱신
    
    Remote 환경에서 Chracter 변경창이 바로 갱신되지 않는 현상 발생

    * 원인

        1. Server의 PlayerState가 복제되기 전 접근해서 Delegate에 바인드 시도 -> 실패 -> 이후 재시도 과정 없음
        2. Delegate 바인드 실패로 인해 Broadcast가 전달되지 않음
    
    * 문제 해결 
        
        PlayerController에 Slot변경을 감지하는 핸들러를 받으면 `AddUniqueDynamic`으로 재바인딩 시도 하는 과정을 추가
