<h3>Unreal 심화 라이브 세션</h3>
<h4>인터페이스</h4>

* UE에서는 다중 상속 미지원

* Interface

    서로 다른 클래스들이 공통된 함수를 갖도록 강제하거나, 공통된 방식으로 호출하기 위해 사용하는 구조
    * UInterface

        언리얼 엔진의 리플렉션 시스템에 인터페이스의 존재를 알리기 위한 클래스

    * IInterface

        실제로 함수를 선언하고 Cpp 클래스가 상속받아 구현하는 클래스

* BlueprintNativeEvent 기반 함수 처리

    * UFUNCTION(BlueprintNativeEvent)

        Cpp에서도 기본 구현을 가질 수 있고, Blueprint에서도 오버라이드할 수 있는 이벤트 함수로 등록하는 지정자

    * `함수명_Implementation`

        BlueprintNativeEvent로 선언된 함수의 Cpp 기본 구현을 작성하는 함수

        * 함수명()을 구현하는 것이 아닌 `함수명_Implementation()`을 구현해야함

    * `Execute_함수명`

        외부에서 인터페이스 함수를 안전하게 호출하기 위해 언리얼이 자동 생성하는 정적 호출 래퍼 함수

        * Blueprint / Cpp 구현을 모두 포함해서 호출하기 위해 사용

<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>NetRole</h4>

* NetRole

    Actor가 현재 실행중인 머신에서 네트워크상 어떤 권한/역할을 가지는지 나타내는 값

    * 종류
        * `None`: 해당 Actor가 네트워크로 Replication되지 않는 상태(Role이 없음)
        * `Authority`: 해당 Actor의 권위 있는 원본 상태를 가진 역할

            중요한 Gameplay 로직을 수행

            > GameMode
        * `Autonomous Proxy`: 서버에 있는 Authority Actor의 소유 클라이언트 측 복제본(주로 Pawn)

            서버 상태를 수신하면서도, 입력이나 Server RPC 등을 통해 서버에 요청을 보낼 수 있음
            > 내가 조종하는 Pawn / Character
        * `Simulated Proxy`: 서버의 Authority Actor의 비소유 클라이언트 측 복제본

            서버에서 복제된 상태를 받아 로컬에서 보간/시뮬레이션함
            > 다른 플레이어의 Character

    * Local Role: 현재 실행 환경에서 이 Actor가 가지는 Role
    * Remote Role: 다른 네트워크 실 실행 환경에서 이 Actor가 가지는 Role

    * 관련 함수
        * `AActor::HasAuthority()`: 현재 Actor의 Local Role이 Authority 인지 확인하는 함수
        * `AController::IsLocalController()`: 이 Controller가 로컬Controller인지 확인하는 함수
        * `APawn::IsLocallyControlled()`: Pawn이 로컬 Controller에 의해 조종되고 있는지 확인
    
    * 특징
        1. GameMode는 서버에만 존재하므로 HasAuthority() 체크를 할 필요가 적음
        2. Pawn / Character는 위치에 따라 Role이 달라진다
            * 서버 - Authority
            * 소유 클라이언트 - Autonomous Proxy
            * 다른 클라이언트 - Simulated Proxy
        3. UI 관련 로직(로컬 표시 로직)은 클라이언트에서 처리함
