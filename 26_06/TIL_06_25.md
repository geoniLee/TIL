<h3>GAS R&D</h3>

GAS(Gameplay Ability System): 캐릭터의 상태 변화를 관리하는 전체 구조

<details>
<summary>사용처</summary>

* 공격·스킬·패시브 능력
* 체력·마나·공격력 같은 능력치
* 피해·회복·버프·디버프
* 스킬 사용 조건, 비용, 쿨다운
* 상태 제어
* 애니메이션·이펙트·사운드
* 멀티플레이 복제와 클라이언트 예측
</details>

* 핵심 구성 요소 
    * Ability System Component

        다른 GAS 요소를 연결하는 중앙 관리자
        
        <details>
        <summary>관리하는 정보</summary>
        * 소유한 GamePlayAbility
        * 적용중인 Gameplay Effect
        * Gameplay Attribute
        * Gameplay Tag
        * Gameplay Cue
        * 어빌리티 활성화 및 취소
        * 서버 클라이언트 간 복제와 예측
        </details>
    
    * GameplayAbility

        캐릭터가 수행할 수 있는 행동
        <details>
        <summary>조정 대상</summary>
        * 실행가능 여부
        * 입력 처리
        * 비용 지불
        * 쿨다운 적용
        * 애니메이션 실행
        * 타겟 탐색
        * Gameplay Effect 적용
        * Gameplay Cue 실행
        * 종료와 취소
        </details>

        * TryActivateAbility(): 실행 가능 여부 확인
        * ActivateAbility(): 실행
        * EndAbility(): 어빌리티 작업이 끝나면 호출
    
    * Ability Task

        Ability 안에서 일정 시간동안 비동기적으로 진행되는 작업
    
    * Attribute

        캐릭터의 수치 상태
        
        ---
        * Base Value: 기본 값
        * Current Value: 적용 중인 Effect까지 반영된 값

    * AttributeSet

        여러 Attribute를 모아놓은 클래스
        * 역할
            * Attribute 값 변경 감시
            * 최대/최소 제한
            * Effect 적용 전후 처리
            * 피해량과 최종 체력 분리
            * Attrbute 복제

    * GameplayEffect

        Attrivute나 Tag를 변경하는 데이터 정의

        * 유형
            * Instant: 즉시 실행되고 종료
            * Duration: 일정 시간 유지
            * Infinite: 제거될 때까지 유지
            * Periodic: 일정 주기로 반복 실행

    * Gameplay Tag

        계층 구조를 가진 식별자
        > Ability가 활성화 될 때 Tag를 부여하거나, 특정 Tag를 가진 Ability를 차단·취소 가능

    * Gameplay Cue

        GAS에서 시각적/청각적 표현을 담당함

        * 이벤트
            * OnActive
            * WhileActive
            * Removed
            * Excuted
        
        > 핵심 게임 로직을 넣지 않음

* 실행 흐름
    * 싱글 플레이
        1. 플레이어 입력
        2. ASC가 GameplayAbility 활성화 시동
        3. 실행조건 검사
        4. Gameplay Ability 활성화
        5. Gameplay Effect 적용
        6. Ability Task로 시전 몽타주 실행
        7. 타겟에게 Gameplay Effect 적용
        8. 대상의 Health Attribute 감소
        9. GameplayCue로 파티클/사운드 실행
        10. EndAbility

    * 멀티 플레이
        1. 플레이어 입력 - (Client)
        2. 클라이언트 ASC가 Ability 활성화 시도 - (Client)
        3. 클라이언트에서 실행 조건을 우선 검사 - (Client)
        4. Ability를 예측 실행 - (Client)
        5. 마나 비용/쿨다운을 예측 적용 - (Client)
        6.  몽타주와 Gameplay Cue를 즉시 실행 - (Client)
        7. 서버에 Ability 활성화 요청
        8. 서버 ASC가 활성화 요청 수신 - (Server)
        9. 서버에서 실행 조건을 다시 검사 - (Server)
        10. 서버가 Ability 활성화를 승인 또는 거부
