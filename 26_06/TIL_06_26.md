<h3>언리얼 심화 라이브 세션</h3>
<h4>GAS</h4>

* Gameplay Effect

    Attribute 변경, Gameplay Tag 부여, Gameplay Cue 실행 등의 효과를 정의하는 데이터

* Duration Policy
    * Instant
        * 적용되는 순간 효과를 실행하고 종료
    * Has Duration
        * 지정된 시간동안 효과 유지
    * Infinite
        * 명시적으로 제거될 때까지 유지

* Duration Magnitude
    
    Has Duration일 때 효과의 유지 시간을 결정
    > Scalable Float Magnitude를 사용하면 기본값이나 GE 레벨에 따른 Curve 값으로 시간을 계산할 수 있음

* Period
    
    효과가 유이되는 동안 Modifier나 Execution을 반복 실행하는 간격
---
* Modifier

    특정 Attribute를 어떤 방식으로 변경할지 정의함

    * 구성요소
        * Attribite : 변경할 대상
        * Modifier Operation: 실제 변경량을 계산하는 방법
        <details><summary>Modifier Operation</summary>
        
        * Add: 더하기
        * Multiply: 곱하기
        * Divide: 나누기
        * Override: 기존 값을 새 값으로 덮어쓰기 
        </details>

    * Magnitude 계산 방식
        * Scaleable Float: 고정된 값 또는 GE 레벨에 따른 Curve 값 사용
        * Attribute Based: Source 또는 Target의 Attribute를 기반으로 계산
        * Custom Calculation Clss: `UGameplayModMagnitudeCalculation`을 상속한 클래스로 계산
        * Set By Caller: GE 에셋에 값을 고정하지 않고 실행 시점에 외부에서 전달
---
* Gameplay Effect

    효과의 원본 정의 또는 템플릿

* Gameplay Effect Spec

    실제로 GE를 적용하기 위해 생성된 런타임 적용 정보

--- 

* ASC에서 Gameplay Effect 적용
    * ApplyGameplayEffectToSelf: 현재 ASC 자신에게 GE 적용

        GE 클래스, 레벨, Effect Context 등을 전달하면 내부적으로 Spec을 만들어 적용하는 편의 함수

    * ApplyGameplayEffectToTarget: 다른 ASC에 GE 적용

        Source ASC가 Target ASC에 효과를 적용

    * ApplyGameplayEffectSpecToSelf: 미리 생성한 GameplayEffectSpec을 자신의 ASC에 적용

    * ApplyGameplayEffectSpecToTarget: 미리 생성한 GameplayEffectSpec을 Target ASC에 적용

        Source ASC에서 준비한 Spec을 다른 ASC에 적용

* CommitAbility

    Ability의 실행을 최종 확정함

    * 수행 작업
        * 비용 지불 가능 여부 감시
        * 쿨다운 적용 가능 여부 검사
        * Cost GameplayEffect 적용
        * Cooldown Gameplay Effect 적용

    * 일부 처리
        * CommutCheck: 비용과 쿨다운 조건 검사
        * CommitCost: 비용만 적용
        * CommitCooldown: 쿨다운만 적용

* WaitTargetData

    Gameplay Ability에서 타겟 선택 결과가 준비될 때까지 기다리는 Ability Task

* Confirmatio Type
    * Instant: 타겟을 찾는 즉시 확정
    * User Confirmed: 타겟 확인 -> 사용자 확인 입력 대기 -> 확정 후 Target Data 전달
    * Custom: Target Actor 또는 Ability의 커스텀 로직이 타겟 데이터의 준비 시점을 직접 결정
    * Custom Multi: 커스텀 로직이 확정 시점을 결정하지만, 하나의 WaitTargetData Task에서 여러번 TaergetData를 반환하는 방식에 사용
