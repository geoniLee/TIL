<h3>알고리즘 문제 풀이</h3>
<h4>오픈 채팅방</h4>

* 알고리즘
    * map
    * queue
    * 문자열 파싱

* 아이디어
    1. 주어진 문자를 공백을 기준으로 나누어 Command, Uid로 저장
    2. Command가 Leave가 아닌 경우 Uid 뒤쪽의 Name을 찾아 map에 갱신
    3. Command가 Change가 아닌 경우 queue에 Command, uid 형식으로 push
    4. queue의 값을 하나씩 꺼내며 Command와 map[uid]의 값를 확인하여 출력할 메시지를 answer에 push

<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>언리얼 엔진 소스코드 빌드</h4>

* Build

    프로젝트의 Cpp 코드를 컴파일·링크하여 실행 바이너리를 생성하는 과정

* Cook

    게임에서 사용하는 에셋을 대상 플랫폼에서 사용할 수 있는 형태로 변환하는 과정

* Stage
    실행 파일과 Cook된 콘텐츠 등 배포에 필요한 파일을 임시 배포 디렉터리에 모으는 과정

* Package
    
    Stage된 결과물을 실제 실행 배포 가능한 형태로 구성하는 과정

* 소스 빌드 엔진

    언리얼 엔진 소스코드를 직접 컴파일한 엔진
    > Client·Server 타깃을 빌드할 때 사용함

* Launcher 엔진

    Epic Games Launcher에서 제공하는 사전 컴파일된 엔진
    > 일반적인 Game 타깃 패키징은 가능하지만, Client·Server 타깃 빌드에서는 적합하지 않음
---
<h4>패키징</h4>

* Build Target

    어떤 용도의 실행 파일을 생성할지 정의함

* Client Target

    서버 전용 코드를 제외한 클라이언트 실행 파일을 생성함

* Server Target

    렌더링과 입력이 필요없는 데디케이티드 서버 실행 파일을 생성

* 주의점
    * 클라이언트와 서버는 서로 역할이 다르므로 각각 별도의 타깃과 패키지로 생성할 수 있음
    * 패키징할 때 포함되지 않은 맵과 에셋은 실행 환경에서 사용할 수 없음
