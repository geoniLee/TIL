<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>NetMode, NetConnection, NetDriver</h4>

* NetMode
    
    해당 게임 프로세스가 네트워크 상에서 어떤 역할을 하고 있는지를 의미(월드의 속성) 

    * 종류 
        * NM_StandAlone

            원격 클라이언트의 연결을 허용하지 않음
            
            * 서버 측 로직과 클라이언트 측 로직을 모두 실행함
            
            > 싱글 플레이 & 로컬 플레이

        * NM_Client

            네트워크 멀티플레이 세션에서 서버에 연결된 클라이언트

            * 서버 측 로직을 실행하지 않으며, 서버로 부터 복제된 Proxy를 보여줌

        * Listen Server

            네트워크 멀티플레이어 세션을 호스팅하는 서버

            * 원격 클라이언트의 연결을 수락하고, 로컬 플레이어를 서버에 직접 배치
                
                (서버 자신도 게임에 참가)

            > 캐주얼 협동 & 경쟁 멀티 플레이어

        * Dedicated Server

            네트워크 멀티플레이 세션을 호스팅하는 서버로 실행

            * 원격 클라이언트의 연결은 허용하지만 로컬 플레이어는 없음

            > 지속적이고 안전한 대규모 멀티플레이어가 필요한 게임

    * UWorld::InternalGetNetMode
        
        NetDriver가 있으면 NetDriver 기준으로 NetMode을 판단한다<br>
        단, 클라리언트 전용 실행이면 NM_Client를 우선 반환할 수 있다.

* NetDriver
    * UNetDriver::GetNetMode

        * NetDriver가 서버 역할
            * 전용 서버 프로세스 - NM_DedicatedServer
            * 로컬 클라이언트 기능이 있는 서버 - NM_ListenServer
        
        * NetDriver가 서버 역할이 아닌 경우 - NM_Client

    * UNetDriver::IsServer

        ServerConnection의 존재 여부로 현재 NetDriver가 서버 역할인지 판단한다.
        
* NetConnection

    UNetConnection은 서버와 클라이언트 사이의 연결 하나를 표현하는 객체로, UNetDriver는 이 UNetConnection들을 소유하고 관리한다.
    * 작동 흐름
        1. 서버와 클라이언트 사이에 네트워크 연결이 성립하면 대응하는 `UNetConnection` 객체 생성
        2. 서버 `NetDrive`에 `UNetConnection`이 ClientConnection 배열에 추가 / 클라이언트 `NetDrive`에 `UNetConnection`이 ServerConnection으로 설정
        3. 서버와 클라이언트는 각자의 `UNetConnection`을 통해 네트워크 통신을 처리

    * 특징
        * `UNetDriver`는 생성된 `UNetConnection` 객체를 소유하고 관리함
        * 서버 PC에 생성된 `UNetDriver`는 접속한 클라이언트의 수만큼 `UNetConnection`을 관리함
        * 클라이언트에 생성된 `UNetDriver`는 ServerConnection 하나만 관리

* Ownership

    액터의 Owner 관계를 통해, 해당 액터가 어느 클라이언트 연결에 속하는지 판단하는 개념

    * 특징
        * 각 `ClientConnection`에는 그 연결을 대표하는 `PlayerController`가 존재함
        * Pawn, Weapon 등은 Owner 체인을 따라 `Owning Connection`이 결정됨
        * 액터의 `Owning Connection`은 `GetnetConnection()`으로 확인할 수 있음

* NetDrive

    네트워크 통신에서 로우 레벨 동작들을 관리하는 클래스

    * 특징
        * 멀티 플레이에서만 UWorld::listen() 함수를 통해 UNetDriver 객체가 생성됨
        * 멀티 플레이어 참여하는 각 PC 마다 UNetDriver 객체가 생성됨
