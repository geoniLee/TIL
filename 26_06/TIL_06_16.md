<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>Property Replication</h4>

* NetLoadOnClient

    레벨에 미리 배치된 액터를 클라이언트도 로딩 과정에서 불러올 것인지 결정하는 속성

    * 레벨 배치 액터
        * 서버와 클라이언트가 맵 로딩 과정에서 각각 로드
        * Net Load On Client가 관여

    * 동적 액터
        * 서버가 런타임에 SpawnActor 등으로 생성
        * Replication을 통해 클라이언에 생성 사실이 전달됨

* Replication Notify

    복제된 프로퍼티의 값이 클라이언트에 도착했을 때, 해당 값에 반응하는 함수를 자동으로 실행하는 기능
    * OnRep_
        * 새로운 값이 복제된 시점에 실행
        * 값이 복제된 클라이언트에서 실행
        * 서버에서도 실행되어야 한다면, 명식적으로 호출해야함
    
    * Blueprint RepNotify
        * 서버에서 값이 변경될 때도 실행될 수 있음
        * OnRep_와 호출 규칙이 같지 않음

<h4>Relevancy</h4>
    서버가 특정 액터를 특정 클라이언트에게 복제할 필요가 있는지를 나타내는 기준

* 연관성 기준
    <details><summary>액터 클래스의 연관성을 판단하는 기준</summary>
    
    * Owner: 해당 액터를 소유하고 있는 액터
    * Instigator: 해당 액터에게 영향을 끼친 폰
    * Always Relevant: 해당 액터가 모든 클라이언트에게 항상 연관성이 있게끔 설정할 때 사용
    * NetUseOwnerRelebancy: 오너 액터의 연관성으로 해당 액터의 연관성을 대신할 때 사용
    * OnlyRelevantToOwner: 오너 액터에게만 연관성을 가짐
    * NetCullDistance: 뷰어와의 거리에 따라 연관성 여부르 결정
    </details>

    <details><summary>폰 클래스의 연관성을 판단하는 기준</summary>
    
    * Viewer: 클라이언트 커넥션이 소유하고 있는 플레이어 컨트롤러를 가리킴
    * ViewTarget: 플레이어가 빙의한 폰
    </details>

* IsNetRelevantFor()

    특정 액터를 특정 클라이언트 연결에 복제해야 하는지 판단하는 함수

<h4>NetPriority</h4>

* NeyProiority
    제한된 네트워크 대역폭에서 어떤 액터를 우선 복제할지 결정하는 값

    > 값이 높을수록 먼저 복제될 가능성이 높음, 상대적인 비율이 중요함

    <details><summary>기본값</summary>
    
    * Actor: 1
    * Pawn: 3
    * PlayerController:5
    </details>

* GetNeyPriority: 현재 복제 우선 순위를 계산하는 함수

* 포화 상태
    
    전송할 전체 복제 데이터가 대역 폭을 초과한 상태
    * 우선순위가 높은 액터부터 복제됨
    * 제외된 액터는 시간이 지나며 우선순위가 올라감

<h4>NetDormancy</h4>

* NetDormancy

    액터를 복제 대상 검사에서 제외하여 서버의 네트워크 연산과 대역폭 사용을 줄이는 기능
    
    * 상태
        * DORM_Never: 기본적으로 휴면을 사용하지 않는 상탱
        * DORM_Awake: 휴면 상태가 아니며, 정상적으로 복제됨
        * DORM_Initial: 맵에 미리 배치된 액터가 처음부터 휴면 상태로 시작
        * DORM_DormantAll: 모든 클라이언트 연결에 대해 휴면 상태
        * DORM_DormantPartial: 일부 연결에 대해서만 휴면

    * 휴면 해제
        * SetnetDormancy(DORM_Awake): 액터를 깨워 지속적으로 정상 복제
        * FlushNetDormancy(): 휴면 상태를 유지하면서 최소 한 번 이상 변경내용 복제

* Conditional Property Replication

    등록된 프로퍼티에 추가적인 복제 조건을 지정하는 기능(한 번 등록하면, 해당 객체의 수명동안 복제 목록에서 해제할 수 없음)

    * COND_InitialOnly: 클라이언트에 액터의 초기 상태를 전송할 때만 복제를 시도
