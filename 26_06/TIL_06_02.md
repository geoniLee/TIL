<h4>기지국 설치</h4>

* 알고리즘
    * 그리디

* 아이디어
    1. 설치된 기지국이 정렬된 상태로 주어지므로 이 범위를 활용
    2. 각 기지국을 순회하며, 기지국의 왼쪽 끝을 설치 오른쪽 값으로 초기화
    3. 이 오른쪽 값이 왼쪽 값과 같거나 작으면 전파가 전달되는 구간이므로 설치하지 않음
        1. 오른쪽 값이 왼쪽 값보다 크면 (r-l + (2*w)) / (2 * w + 1)을 더함
    4. 검사 범위의 왼쪽 끝을 기지국의 오른쪽 범위로 초기화함

<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>서버의 종류와 데디케이티트 서버</h4>

* 서버
    * P2P Server(Pear to Pear Server): 각각의 컴퓨터가 클라이언트이자 서버
    * Listen Server : 클라이언트이자 서버인 방장(Host)가 있고, 나머지 참가자(Guest)는 모두 클라이언트 역할을 함
    * Dedicated Server: 서버를 담당하는 컴퓨터가 따로 존재함

* 데디케이티드 서버
    * 실행 흐름도
        1. PIE나 Server.exe를 실행하는 식으로 서버 프로세스 실행
            * Open {Level이름}?Listen 명령어가 인자로 전달
            * 이 때 Socket이 생성되며 다른 PC가 접속 가능하게끔 함
            * Level에는 WorldSettings 속성이 있고, WorldSettings에서 GameMode와 GameState 액터를 생성합니다.
            > GameMode 액터는 Server에만 존재함
        2. 클라이언트는 서버의 IP 주소와 포트번호로 접속 시도, 서버는 클라이언트에게 Level 정보를 넘김
        3. Level을 여는데 성공하면, 클라이언트 전용 PlayerState, PlayerController, PlayerCharacter가 서버에 생성되고 이를 클라이언트에 복제됨
        4. 다른 클라이언트가 서버에 접속하면 동일하게 2~3의 과정을 합니다
        5. 다른 클라이언트가 접속하면 클라이언트 간의 PlayerState와 PlayerCharacter도 복제되면서 서로가 보이게 됨
    * 특징
        * 클라이언트와 클라이언트 간의 통신이 불가능
        * RPC나 프로퍼티 레플리게이션에도 영향을 끼침

<h4>멀티플레이 실습 환경 설정</h4>

* EditorPreferences
    * Allow Late joining: 실행 중 Add another cliner 버튼으로 클라이언트 추가 가능
    * Always on top: 새 클라이언트 화면이 항상 위에 뜸
    * Multiplayer Options
        * Launch Seperate Server: 서버를 따로 띄워 줌
        > 미 체크 시 에디터 화면이 클라이언트이자 서버임
        * Run Under One Process: 체크 시 같은 프로세스 내라서 공유하는 에셋이 생김
        > 성능상의 이점이 있으며, 체크하지 않으면 서버와 각 클라이언트들이 다른 프로세스로 분기됨
        <br>
        > 추후 디버깅 시 체크 후 로깅을 찍어볼 수 있음
