<h3>알고리즘 문제 풀이</h3>
<h4>2 x n 타일링</h4>

* 알고리즘
    * DP(다이나믹 프로그래밍)

* 아이디어
    1. n의 길이를 가진 타일의 경우의 수는 n-2의 경우의 수 + n-1의 경우의 수
    2. 기저 조건으로 n이 0일때와 1일 때 1로 초기화
    3. 2부터 시작해서 n까지 dp값 초기화
    4. dp[n] 반환

---
<h4>파일명 정렬</h4>

* 알고리즘
    * 문자열
    * 우선순위 큐

* 아이디어
    1. 각 file을 순회하며, 숫자가 나오기 전까지를 Head, 숫자 이후 다시 문자가 나오기 전까지Num에 저장
    2. Priority_queue을 greater을 사용해서 정렬
    3. pq에 Head, stoi(num), idx을 넣어 정렬하도록 함
    4. pq의 값을 하나씩 꺼내 반환

<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>액터 컴포넌트 동기화</h4>

* 액터 컴포넌트 동기화
    * 액터 컴포넌트(서브 오브젝트)는 독립적으로 복제되는 것이 아니라 소유 액터의 복제 과정에 포함되어 동기화 됨
    <details><summary>조건</summary>

    * 소유 액터가 Replication을 사용
    * 컴포넌트가 Replication을 사용
    * 복제할 프로퍼티를 등록
    </details>
    * 서버에서 변경된 컴포넌트 프로퍼티는 클라이언트에 복제되며, RepNotify를 통해 UI 등의 로컬 상태를 갱신할 수 없음
    * 컴포넌트에는 독립적인 Role이 없으며, Relevancy·Priorit·Ownership 등은 소유 액터를 따름
    
* 컴포넌트 초기화 순서
    1. InitializeComponent()
    2. ReadyForReplication()
    3. BeginPlay()

* COND_OwnerOnly

    해당 프로퍼티를 그 액터의 OwningClient에 복제하는 조건
    > 다른 클라이언트에게는 복제하지 않음
