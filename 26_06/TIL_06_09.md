<h3>Unreal Engine 멀티플레이어 게임 개발</h3>
<h4>Remote Procedure Call 기초</h4>

* RPC(Remote Procedure Call)

    한 머신에서 호출한 함수를 네크워크를 통해 다른 머신에서 실행되도록 요청하는 통신 방식

    > 게임에 큰 영향을 끼치지 않는 일시적인 효과들에 사용(코스메틱 - 사운드 / 파티클)

    * 특징
        * RPC는 단방향 함수 호출이므로 일반 함수처럼 반환값을 직접 받을 수 없음
        * RPC는 Replicate되는 Actor / ActorComponent를 통해 호출해야 함

    * Call

        정적인 함수 호출 <br> 
        호출 대상이 명확인 정해져 있는 경우(Direct)

        > 일반 함수, 정적 바인된 함수, 멤버 함수 직접 호출

    * Invoke

        동적인 실행 요청<br>
        런타임에 호출 대상이나 실행 위치가 결정됨(InDirect)

        > 함수 포인터, 델리게이트, 가상 함수 호출, 동적 바인딩, RPC

* ActorOwnership

    * Actor의 최종 Owner가 내 `Local PlayerController`: Client-Owned Actor

        해당 Client가 이 Actor를 통해 ServerRPC를 호출 가능

    * Actor의 최종 Owner가 다른 Client의 `Local PlayerController`: Owned by Different Client

        현재 Client는 이 Actor를 통해 Server RPC를 호출할 권한이 없음

    * 서버에서 스폰되었고, 특정 Client의 PlayerController를 Owner로 가지지 않음: Server-Owned Actor / No Owning Client

        특정 Client로 향하는 Client RPC 대상이 명확하지 않음

* 키워드
    
    해당 원격 PC에서 RPC를 실행시켜 달라는 요청

    * NetMulticast: 서버에서 호출 했을 때, 서버와 해당 Actor가 Relevant한 클라이언트들에서 RPC를 실행시켜 주세요
    * Server: 서버에서 해당 RPC를 실행시켜 주세요
    * Client: 해당 Actor의 Owning Client에서 RPC를 실행시켜 주세요

    * WithValidation

        Server RPC가 실제로 실행되기 전에, 서버에서 해당 요청을 받아들일지 검사

        * 이 키워드가 붙은 RPC를 구현할 때 `_Implementation()` 함수와 `_Validate()` 함수로 나누어짐
        * `_Validate()` 함수는 해당 RPC 요청이 유효한지 검사한 후 실행

    * Unreliable VS Reliable

        기본적으로 Unreliable을 기준으로 동작

        * Unreliable
            목적지에 도착하거나 실행된다는 보장이 없는 RPC

            > 코스메틱(이펙트, 사운드 등)처럼 게임에 큰 영향이 없는 로직

        * Reliable
            목적지에 도착할 때까지 큐에 남아 재전송되는 RPC

            > 게임에 큰 영향을 주는 로직(충돌, 데미지 스폰)

<h4>멀티플레이 채팅 구현</h4>

* 데디케이트 서버 구조
    
    * 클라이언트끼리는 직접 통신하지 않음

    > Client -> Server -> Client

* 채팅 메시지 전파

    * 클라이언트가 입력한 메시지는 Server RPC로 서버에 전달됨
    * 서버는 모든 PlayerController를 순회하며 각 클라이언트에게 ClientRPC를 호출한다

* 접속 알림
    
    * 접속 감지는 GameMode의 OnPostLogin에서 한다.
    * 전체 클라이언트 알림은 GameState의 NetMulticast RPC로 처리한다.

<h4>Property Replication</h4>

* 레플리케이션

    생성된 액터의 정보를 네트워크 내 다른 클라이언트에게 복제하는 작업

    * 서버 - 클라이언트 모델에선 서버에서 클라이언트로 복제되는 것이 기본 규칙

    * 방법
        1. RPC
        2. Property Replication

* Property Replication

    우리가 원하는 속성만을 다른 클라이언트에게 복제하는 것

    * 설정 방법
        1. 액터의 `Replicates` 속성을 true로 설정
        2. 네트워크로 복제할 액터의 속성을 키워드로 지정 (UPROPERTY 등에 추가)
        3. `GetLifetimeReplicatedProps()` 함수에 복제할 속성 추가
            * `#include "Net/UnrealNetwork.h"`추가
            * `DOREPLIFETIME` 매크로를 사용해 복제할 속성 명시
