<h4>해시와 문자열 처리</h4>

* 해시<br>
키를 사용하여 값을 빠르게 접근할 수 있도록 데이터를 구성하는 자료구조입니다.
    * 주의점
        1. 해시 함수가 반환하는 인덱스는 반드시 해시 테이블의 크기보다 작아야함
        2. 충돌(다른 키에 대해 동일한 값 반환)이 최대한 적게 발생하도록 설계해야함

    * 나눗셈법 - 키를 소수로 나눈 나머지 활용
    * 곱셈법 -실수(황금비 등의 무리수)를 사용하여 해시 인덱스를 계산
    * 문자열 해싱 - 문자열을 정수로 반환하여 해시값을 계산(다항식 해싱)
        ```
        // p =31
        hash(s) = (s[0]+ s[1]*p + s[2]*p^2 + ... +s[n-1]*p^(n-1))mod m
        ```

* 해시 충돌
    * 체이닝 - 해시 충돌이 발생했을 때, 해당 해시 버킷에 값을 연결 리스트 형태로 저장
        * 구현이 간단함
        * 충돌이 발생할 때마다 링크들의 리스트 길이가 길어져 성능이 저하됩니다.
    * 개방 주소법 - 충돌 시, 다음 빈자리를 찾아 값을 저장합니다.
        * 별도의 연결리스트나 외부 메모리를 사용하지 않으므로, 추가 메모리 할당이 없습니다.
        * 충돌이 많아지면 클러스터링이 발생(군집이 형성되어 성능 저하)
    * 이중 해싱 - 해시 함수를 2개 사용
        * 선형탐사(개방 주소법)이나 이차 탐사에 비해 클러스터링 현상이 적음
        * 충돌 처리 효율이 높음

<h4>동적 계산법</h4>

* 동적 계산법<br>
복잡한 문제를 더 작고 관리하기 쉬운 하위 문제로 분할하여 해결하는 방법론
    * 사용처
        * 최적 부분 구조 - 최적 해결책이 하위 문제들의 최적 해결책으로부터 구성될 수 있음
        * 중복 부분 문제 - 문제의 해결 과정에서 동일한 하위 문제가 여러번 반복적으로 나타나는 경우

    * 해결 과정
        1. 최적 부분 구조 및 중복 부분 문제인지 확인
        2. 기저 조건 확인
        3. 점화식 수립

* 메모이제이션<br>
한 번 계산한 결과를 저장해두고, 이후 동일한 문제가 다시 등장했을때 재사용하는 기법

* 최장 증가 부분 수열(LIS)<br>
부분 수열 중에서 원소들이 오름차순으로 증가하고, 그 길이가 가장 긴 수열
    * 부분 수열 - 일부 원소를 순서를 유지한채 선택하여 만든 새로운 수열
