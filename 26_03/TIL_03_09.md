<h3>게임 개발자를 위한 C++ 문법</h3>
<h4>템플릿</h4>

* 함수 오버로딩<br>
    매개 변수의 개수나 자료형을 다르게 하여, 동일한 이름의 함수를 여러 개 재정의 하는 것
    - **네임 맹글밍**: 함수 이름 구분을 위해 내부적으로 고유한 이름을 부여하는 것

    **오버로딩 되지 않는 경우**
    - 두 개 이상의 오버로딩된 함수가 호출 후보가 되는 경우
    - 디폴트 매개변수로 인해 함수 호출 형태가 중복되는 경우
    - 매개변수의 타입만 포인터와 배열로 다른 경우
    - 함수의 반환 타입만 다른 경우

    **함수 오버로딩의 순서**
    1. 정확한 매개변수 타입 일치
    2. 타입 승격 변환
        * char/short -> int
        * float -> double
        * bool -> int
    3. 표준 타입 변환
        * int -> double
        * bouble -> float
        * double -> int
    4. 사용자 정의 타입 변환
        * 클래스의 타입 변환 함수
        * 생성자 등을 통해 이뤄지는 변환

* 탬플릿
    > 타입에 관계없이 일반화된 코드를 작성하기 위한 문법>

    ```
    template\<typename T>
    ```
    * T: 어떤 타입이 올지 모르겠으나, 그 타입을 T라고 부름 

    * 클래스도 템플릿으로 사용해 일반화 할 수 있음

<h4>STL 기초</h4>

* STL(Standard Template Library)<br>
    * 컨테이너, 알고리즘, 반복자 등의 요소를 포함합니다.

    * 컨테이너: 데이터를 담는 자료구조(템플릿으로 구현)
        - 메모리 관리를 내부적으로 함
        - 반복자를 제공함

    * vector 
        * 템플릿 클래스로 구현되어 특정 타입에 종속되지 않음
        * 삽입되는 원소 개수에 따라 내부 배열의 크기가 자동으로 조정됨
        * 임의 접근이 가능함


        **벡터의 선언**
        1. 빈 벡터 선언 / 특정 값으로 초기화
            ```
            vector\<type> name;
            
            vector\<type> name(size, value);
            ```
        2. 초기화 리스트를 사용하여 벡터를 선언
            ```
            vector\<type> name = {1,2 ... n};
            ```
        3. 다른 벡터에 복사하거나 대입
        4. 2차원 벡터
            ```
            vector<vector\<type>>> name
            ```

        **벡터의 동작**
        1. push_back: 벡터의 맨 끝에 원소를 추가하는 메서드
        2. pop_back: 벡터의 맨 끝에 원소를 제거하는 메서드
        3. size: 벡터의 크기를 반환
        4. erase: 특정 위치의 원소를 제거하는 함수
            * unique와 주로 같이 사용

    * map<br> 
    특정 키를 사용하여, 값을 검색하는 기능을 제공하는 컨테이너<br>
        **맵의 선언**<br>
        ```
        map<type, type> name;
        ```

        **특성**<br>
        1. 키-값 쌍은 pair<const key, Value> 형태로 저장됨
        2. 키 값을 기준으로 자동으로 정렬됩니다.
        3. 중복된 키값은 허용되지 않습니다.

        * insert: 값을 추가 할 수 있습니다
        ```
        m.insert[key] = value;
        ```
            * map에 key의 값이 없다면 자동으로 생성됩니다.

        * find(): 특정 키가 map에 존재하는 지 확인합니다(1 or 0)
        ```
        m.find(key);
        ```

        * size: 키-쌍 개수를 반환함

        * erase(key): map의 특정 key을 가진 요소만 삭제함

        * clear: 맵에 있는 모든 원소를 삭제합니다.

    * sort<br>
    컨테이너 내부의 데이터를 정렬하는 함수
        * 사용자 정렬 함수를 없이 쓰면, 오름차순으로 정렬됩니다

        ```
        sort(first, last, 사용자 정렬 함수);
        ```
    
    * find<br>
    컨테이너 내부에서 특정 원소를 찾아 해당 원소의 반복자를 반환하는 함수

        ```
        find(first, last, 찾을 값);
        ```

        * 원소를 찾은 경우 해당 원소의 반복자를 반환합니다.
        * 원소를 찾지 못한 경우 last 반복자를 반환합니다

    * 반복자
        * 순방향 반복자: 앞에서 부터 뒤로 순차적으로 순회하는 반복자
            * begin(): 첫번째 원소
            * end(): 마지막 원소 다음
        * 역방향 반복자: 마지막 원소부터 첫번째 원소까지 역순으로 순회할 수 있도록 해주는 반복자
            * rbegin(): 마지막 원소
            * rend(): 첫번째 원소 이전
