<h3>Cpp 코딩테스트 완전 정복</h3>
<h4>트리 및 탐색</h4>

* 트리<br>
데이터를 계층적인 방식으로 조직하고 관리하는 비선형 자료구조
    * 구성 요소
        * 노드: 트리를 구성하는 요소
        * 간선: 노드와 노드 사이를 이어주는 선
        * 레벨: 루트 노드에서 특정 노드까지 거쳐가는 최소한의 간선 수
        * 부모/자식 노드: 서로 부모-자식 관계
        * 형제 노드: 부모가 같은 노드
        * 루트 노드: 노드 중 가장 최상위에 있는 노드
        * 리프 노드: 자식 노드가 없는 노드
        * 차수: 특정 노드에서 아래로 향하는 간선의 개수

    * 순회: 트리에 존재하는 모든 노드를 빠짐없이 한 번씩 방문하는 과정
        * 전위 순회
            > 순회 순서<br>
            > 부모 노드 -> 왼쪽 자식 노드 -> 오른쪽 자식 노드
        * 중위 순회
            > 순회 순서<br>
            > 왼쪽 자식 노드 -> 부모 노드 -> 오른쪽 자식 노드
        * 후위 순회
            > 순회 순서<br>
            > 왼쪽 자식 노드 -> 오른쪽 자식 노드 -> 부모 노드

    * 이진 탐색 트리
        * 규칙
            1. 현재 노드보다 값이 적으면 왼쪽으로 이동한다
            2. 현재 노드보다 값이 크거나 같은면 오른쪽으로 이동한다
        
        * 배열 탐색과의 비교
            * 배열(O(N)) - 한번에 하나의 원소만 제외
            * 이진 탐색 트리(O(log N)) - 한 번의 비교로 서브 트리 전체를 건너뛸 수 있음

<h4>그래프 개념 및 탐색</h4>

* 그래프<br>
유한한 수의 정점과 이들을 연결하는 간선들의 집합으로 구성된 비선형 자료구조
    * 구성요소
        * 노드
        * 간선
        * 가중치

* 인접 행렬 -> 2차원 배열을 이용해 그래츠의 연결관계를 표현하는 방식
    * 메모리 > v * v 
* 인접 리스트 -> 각 노드에 연결된 노드들을 리스트로 저장
    * 메모리 > 연결된 노드 수 만큼 사용

* 인접 행렬 vs 인접 리스트
    * 인접 행렬
        * 단점
            1. 노드 수에 비에 간선 수가 적은 그래프 생성 시 메모리 낭비가 심함
        * 장점
            1. 간선이 있는지 확인하는 연산 = O(1)
            2. 임의 접근을 통해 바로 확인 가능
            3. 구현 난이도 낮음
    * 인접 리스트
        * 단점
            1. 간선의 개수 만큼 시간복잡도가 늘어남
        * 장점
            1. 메모리 낭비가 없음

* 그래프 탐색 
    * 깊이 우선 탐색
        * 기본 동작
            1. 더 이상 방문할 노드가 없으면, 가장 최근에 방문했던 노드로 갑니다
            2. 해당 노드와 연결된 노드 중 방문하지 않은 노드가 있는지 확인 후 '1'을 수행합니다.
            3. 위 과정을 모든 노드를 방문할 때 까지 수행합니다.
        * 재귀 함수 - 함수의 호출 과정과 유사함
        * 스택 - 후입 선출(LIFO)
    * 너비 우선 탐색
        * 기본 동작
            1. 시작 정점에서 방문 가능한 모든 인접 노드를 큐에 넣습니다
            2. 큐에서 노드를 하나씩 꺼내며, 꺼낸 노드와 인접한 노드 중 아직 방문하지 않은 노드를 큐에 넣습니다
            3. 1~2을 반복하며 모든 노드를 탐색합니다.
        * 큐 - 선입선출(FIFO)
    * dfs vs bfs
        * bfs
            * 최단 경로 탐색에 주로 쓰입니다.
            * 검색 대상의 규모가 크지 않거나, 시작 지점으로 부터 원하는 대상이 별로 멀지 않은 경우
        * dfs
            * 경로의 특징을 저장해뒤야 하는 문제(ex. 경로에 같은 숫자가 있으면 안되는 경우 등)
            * 검색 대상 그래프가 큰 경우
