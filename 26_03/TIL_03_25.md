<h3>알고리즘 문제 풀이</h3>
<h4>12849 본대 산책</h4>

* 알고리즘
    * dp(다이나믹 프로그래밍)
    * 그래프 탐색
* 아이디어
    1. dp에 각 노드에서 0까지의 소요되는 최소 시간으로 초기화
    2. 시간 t 가 0~d 까지 방문할 수 있는 경우의 수를 저장
    3. 저장할 때 해당 노드에서 남은 시간 내에 출발지 까지 도착할 수 없는 경우 값을 추가 하지 않음
    4. 시간 t일 때 0번 노드에 있는 값을 출력
* 회고
    1. bfs사용 및 방문 검사를 하지 않아 누락된 값 없이 계산 -> 중복된 값을 반복적으로 연산하게 되어 시간 초과 예상 -> 시간, 해당 노드에 방문 가능한 경우의 수로 변경

<h4>11726 2×n 타일링</h4>

* 알고리즘
    * dp(다이나믹 알고리즘)

* 아이디어
    * 규칙 탐색(n > 2일 때 dp[n] = dp[n-2] + dp[n-1])
    * dp 값이 아직 없으면 dp값을 구하고, 있으면 반환
