<h3>알고리즘 문제 풀이</h3>
<h4>3151 합이 0</h4>

* 알고리즘
    * 정렬
    * 두 포인터
* 아이디어
    1. 정렬 후 첫번째 팀원을 고정한 상태에서 두 포인터로 탐색
* 회고
    * 세 팀원의 합이 0 일때 cnt을 하고 다음으로 넘어 가니 -1,-1,0,0, 1,1 와 같은 경우 놓치는 케이스가 생기게됨
    * 첫번째 팀원 고정, 세번째 팀원의 값을 더했을때 더 이상 합이 0이 나오지 않는 경우 두번째 팀원 인덱스 이동 및 세번째 팀원의 인덱스 초기화
        * 사실상 for문을 세번 써서 구현한 것과 다를 바 없이 시간초과
    * 2, 3 번째 멤버 두 포인터 사용
        1. 합이 0일 때 현재 3번째 멤버의 값이 처음 나오는 위치 찾기
        2. 2번째 멤버의 값과 세번째 멤버의 값이 같다면, 중복된 값의 수 만큼 cnt 증가 시키고 1번째 멤버 인덱스 증가
        3. 2번째 멤버의 값가 세번째 멤버의 값이 같지 않은 경우, 왼쪽의 중복된 값의 수 * 오른쪽 중복된 값의 수를 cnt에 추가하고 그 위치에서 두포인터를 이어서 진행

<h4>재귀 함수</h4>

* 재귀 함수: 자기 자신을 다시 호출하는 방식의 함수
    * 재귀 호출 단계: 자기 자신을 호출 - 호출할 때마다 문제의 크기를 줄여야함
    * 기저 조건: 종료 시킬 조건

* 수학적 귀납법 - 재귀<br>
    Ex. 도미노 효과
    * 첫 번째 도미노가 넘어집니다(기본 단계 / 기저 조건)
    * k번째 도미노가 넘어지면, k+1번째 도미노가 반드시 넘어집니다.(귀납 단계/ 재귀 호출 단계)

* 재귀 함수 - 메모리
    1. 함수 호출 -> 동작하는데 필요한 정보들이 메모리에 저장됩니다
        * 각각의 호출마다 새로운 스택 프레임이 생성되어 스택 메모리에 쌓임
        * 함수가 종료되면, 해당 스택 프레임이 제거
    2. 마지막 함수까지 호출되면, 자신으로 호출했던 위치로 돌아가며 종료
    
* 재귀 함수 - 주의점
    1. 종료 조건을 명확히 하고, 호출 깊이를 적절히 제한하여 메모리 사용을 관리할 것
    2. 함수가 실행된 후 종료되면, 호출 지점으로 돌아가 중단된 지점 다음부터 실행을 이어갑니다.

* 메모이제이션(dp)<br>
한번 계산한 결과를 저장해두고, 이후에는 다시 계산하지 않는 것

<h4>배열</h4>

* 배열: 동일한 데이터 타입의 여러 원소들을 연속된 메모리 공간에 저장하여 관리하는 선형 자료 구조<br><br>
* 특징
    * 동질성: 배열의 원소는 모두 동일한 타입을 가짐
    * 연속된 메모리 할당: 접근 속도가 빠름
    * 인덱스 기반 접근
    <br><br>
    * 시간 복잡도
        * 맨 뒤에 삽입하는 경우 -> O(1)
        * 맨 앞에 삽입하는 경우 -> O(n)

* 배열 선언
    * 1차원
    ```
    data_type array_name[size]

    data_type array_name[size] = {value1, value2, ...valueN}
    ```
    * 2차원
    ```
    data_type array_name[row_size][col_size]

    data_type array_name[row_size][col_size] = {
        {val1, val2, val3, ...valN},
        {valN+1, valN+2, valN+3,...val2N}, 
        ...}
    ```

* 배열 메모리 구조
    * 1차원 - 첫번째 원소부터 차례로, 연속적인 메모리 공간에 저장
    * 2차원 - 1행 부터 차례로 연속적인 메모리 공간에 행우선(row-major) 순서로 저장
